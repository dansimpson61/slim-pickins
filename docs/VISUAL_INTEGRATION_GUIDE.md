## Layer by Layer

### Layer 1: Raw Data (Ruby/Model)

```ruby
# In your app/controller
@revenue = 1234.56
@growth_rate = 0.1234
@user_count = 5432
@created_at = Time.parse("2025-10-16 14:30:00")
```

### Layer 2: Structure (Helpers)

```slim
/ Helpers create the HTML structure
== card "Metrics" do
  .metric
    .label Revenue
    .value [DATA GOES HERE]
  
  .metric
    .label Growth
    .value [DATA GOES HERE]
```

**Helper generates:**
```html
<div class="card">
  <h3>Metrics</h3>
  <div class="metric">
    <div class="label">Revenue</div>
    <div class="value">[DATA GOES HERE]</div>
  </div>
  <div class="metric">
    <div class="label">Growth</div>
    <div class="value">[DATA GOES HERE]</div>
  </div>
</div>
```

### Layer 3: Formatting (Filters - Future)

```slim
/ Filters format the data inline
== card "Metrics" do
  .metric
    .label Revenue
    .value
      currency:
        = @revenue
  
  .metric
    .label Growth
    .value
      percent:
        = @growth_rate
```

**Complete output:**
```html
<div class="card">
  <h3>Metrics</h3>
  <div class="metric">
    <div class="label">Revenue</div>
    <div class="value">$1,234.56</div>
  </div>
  <div class="metric">
    <div class="label">Growth</div>
    <div class="value">12.34%</div>
  </div>
</div>
```

### Current Workaround (v0.1 - No Filters Yet)

```slim
/ Use Ruby helpers for formatting
== card "Metrics" do
  .metric
    .label Revenue
    .value = number_to_currency(@revenue)
  
  .metric
    .label Growth
    .value = "%.2f%%" % (@growth_rate * 100)
```

### âœ… Perfect Balance

```slim

== card "User Profile" do
  
  / Helper creates component
  == avatar @user, size: :large
  
  / Plain semantic HTML
  h3 = @user.name

  dl.user-details

        = @user.created_at
```

**Breakdown:**
- 3 Helpers: `card`, `avatar`, `action_button`
- 2 Filters: `date:`, `time_ago:`
- Plain HTML: `h3`, `dl`, `dt`, `dd`
- Ruby: `= @user.name`, `= @user.email`
